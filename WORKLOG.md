## 2026-05-10 — point config.yaml at mimic-only 03_*.csv splits

**Goal.** Make `camchex/config.yaml` use the full MIMIC dataset by default, since `data/data-camchex/` no longer contains plain `03_train/development/test.csv` — only the source-prefixed `03_mimic_*.csv` and `03_kaggle_*.csv` variants exist after the multi-source filtering refactor (commit d5e5d70).

**Changes.**
- `camchex/config.yaml:38` — `PredictionWriter.pred_df_path` → `../data/data-camchex/03_mimic_test.csv`.
- `camchex/config.yaml:124-126` — `train_df_path`, `devel_df_path`, `pred_df_path` under `datamodule_cfg` → `03_mimic_{train,development,test}.csv`.

**Reasoning.** `03_filter_existing_images.py` now emits per-source CSVs (`03_mimic_*` and `03_kaggle_*`) instead of one combined `03_*`. The old config paths silently 404 at training start. User asked for the *full mimic* set as the default; kaggle is the smaller subset, so mimic is the natural default. Picked path-replacement over introducing a YAML anchor / variable since the file has no other interpolation and a flat literal change keeps `config.local.yaml` overrides obvious.

**Gotchas.** There are *two* places the test CSV is referenced — once inside the `PredictionWriter` callback (line 38) and once under `datamodule_cfg` (line 126). They must agree, otherwise predictions are written for one split while the dataloader serves another. Both were updated.

**Follow-ups.** If a machine wants kaggle by default, override the three `*_df_path` fields in `config.local.yaml` rather than editing `config.yaml`. Worth confirming `03_mimic_*.csv` exist on ict14/macmini/richmadam after next rsync — they're regenerated by step 3, not synced.
