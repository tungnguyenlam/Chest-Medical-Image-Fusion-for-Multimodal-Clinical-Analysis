
\section{Methodology}
\label{sec:methodology}

We adapt the CaMCheX multimodal architecture to study whether clinical context and prior-study information improve long-tailed chest X-ray classification. The final model uses current chest X-ray images, pre-interpretation clinical text, structured vital signs, and prior-study context. This section describes only the final architecture used in the main experiments. Earlier design choices are discussed only as ablations in Section~\ref{sec:results}.

\subsection{Problem Formulation}
\label{subsec:method_problem}

The task is multi-label chest X-ray classification with multimodal clinical context. For each current study, the input consists of one or more chest X-ray images, clinical indication text, structured vital signs, and, when available, information from a previous study. The model predicts a binary label vector for the presence or absence of each thoracic finding.

Let each sample be represented as:
\begin{equation}
    x_i = (I_i, T_i, V_i, M_i, P_i)
\end{equation}
where \(I_i\) is the set of current chest X-ray views, \(T_i\) is the clinical indication text, \(V_i\) is the vital-sign vector, \(M_i\) is the missingness mask, and \(P_i\) is optional prior-study information. The target label is a multi-hot vector representing the presence of findings:
\begin{equation}
    y_i \in \{0,1\}^{C}
\end{equation}
where \(C\) is the number of pathology labels. The model is trained to learn a parameterized function \(f_\theta\) that produces the class probability predictions:
\begin{equation}
    \hat{y}_i = f_\theta(x_i) \in [0,1]^C
\end{equation}
where each output dimension \(\hat{y}_{ic}\) corresponds to the predicted probability of the \(c\)-th pathology label.

\subsection{Overview of the Proposed Model}
\label{subsec:method_overview}

Our model encodes each modality separately and fuses the resulting tokens with a transformer \cite{vaswani2017attention}. First, we process each input modality with a dedicated encoder. Chest X-ray images are encoded using a lightweight visual backbone. Clinical indication text is encoded using a radiology-domain language model. Vital signs are normalized and projected through a small multilayer perceptron (MLP) together with their missingness mask. When a prior study exists, we encode it separately and use it as temporal context.

After encoding, all modality features are projected into a shared latent dimension. Current-study information forms the target stream, while prior-study information forms a memory stream. The fusion module first models interactions among current-study tokens and then allows the current study to attend to prior-study context through cross-attention. This design keeps the current examination as the main prediction target while allowing historical information to support interpretation.

The final fused representation is passed to a multi-label classification head, which outputs one logit for each target finding. The model is trained using an imbalance-aware loss function suitable for long-tailed multi-label classification.

\subsection{Data Preprocessing}
\label{subsec:method_preprocessing}

Before encoding the inputs, we standardize the data, handle missing values, and remove leakage-prone fields:

\textbf{Image Preprocessing}: Chest X-ray images are resized to a resolution of $512 \times 512$ pixels. The final model uses a three-channel chest X-ray input. The three channels are the raw grayscale image, a Contrast Limited Adaptive Histogram Equalization (CLAHE) enhanced image \cite{zuiderveld1994clahe}, and a histogram-equalized image. This keeps the input compatible with standard image backbones while giving the model additional contrast information. A plain three-channel grayscale input is used only as an ablation in Section~\ref{sec:results}. Pixel values are z-score normalized using the mean and standard deviation computed over the training subset.

\textbf{View Handling}: Studies with multiple radiographic views are limited to a maximum of $4$ views. Views are classified based on their metadata label (\texttt{ViewPosition}): AP, PA, or FRONTAL are mapped to a frontal code ($1$), LATERAL or LL are mapped to a lateral code ($2$), and any other view is coded as $0$. Studies with fewer than $4$ views are zero-padded, and a binary view attention mask is generated so that the transformer fusion layer ignores the padding tokens.

\textbf{Text Tokenization}: Clinical indication texts (and history if present) are parsed and concatenated. Text is tokenized using the tokenizer of the domain-specific \texttt{BiomedVLP-CXR-BERT-}\-\texttt{specialized} model \cite{boecking2022biovil}. The text sequences are padded or truncated to a maximum length of 384 tokens.

\textbf{Vital-Sign Normalization and Outlier Handling}: The structured vital signs vector consists of six numerical parameters (temperature, heart rate, respiratory rate, oxygen saturation, systolic blood pressure, diastolic blood pressure) plus one demographic feature (gender). To handle extreme outliers, values are clipped to physiologically plausible ranges (e.g., temperatures between $85^\circ\text{F}$ and $110^\circ\text{F}$, oxygen saturation capped at $100\%$). Z-score normalization is then performed. For each patient study \(i\) and vital variable \(j\), the normalized value \(\tilde{v}_{ij}\) is formulated as:
\begin{equation}
    \tilde{v}_{ij} =
    \begin{cases}
        \frac{v_{ij} - \mu_j}{\sigma_j}, & \text{if } v_{ij} \text{ is observed}, \\
        0, & \text{if } v_{ij} \text{ is missing},
    \end{cases}
\end{equation}
where \(\mu_j\) and \(\sigma_j\) are the mean and standard deviation of variable \(j\) computed strictly from the training set only to prevent evaluation leakage.

\textbf{Missing Value Imputation}: Missing vital signs are imputed with a neutral z-score value of \(0.0\). To inform the model of data availability, a binary missingness mask \(m_{ij}\) is constructed as:
\begin{equation}
    m_{ij} =
    \begin{cases}
        1, & \text{if } v_{ij} \text{ is observed}, \\
        0, & \text{if } v_{ij} \text{ is missing}.
    \end{cases}
\end{equation}
This mask vector \(M_i = [m_{i1}, \dots, m_{iD}]^\top\) is concatenated to the normalized vital-sign vector \(\tilde{V}_i = [\tilde{v}_{i1}, \dots, \tilde{v}_{iD}]^\top\), enabling the subsequent vital-sign projection network to dynamically adjust its weighting based on data availability.

\textbf{Prior-Study Selection}: To compile longitudinal patient history, studies are sorted chronologically per patient. We use a chronological shift to select the immediately previous study as the prior. This chronological constraint prevents the model from using future information.

\textbf{Label Construction}: The binary targets correspond to the $26$ long-tailed pathologies defined by the CXR-LT-2023 dataset. Label vectors are represented as $26$-dimensional multi-hot arrays.

\subsection{Chest X-Ray Image Encoder}
\label{subsec:method_image_encoder}

The image encoder extracts visual features from the current chest X-ray study. Because chest X-ray studies may contain different view positions, the model keeps view-position information and masks missing view slots. Each available view is passed through a lightweight ConvNeXt-V2 visual backbone \cite{woo2023convnextv2} to obtain spatial feature maps. These feature maps are then projected to the transformer latent dimension using a convolutional projection layer. Formally, for a set of image views \(I_i\), the image representation \(H_i^{img}\) is extracted as:
\begin{equation}
    H_i^{img} = E_{img}(I_i) \in \mathbb{R}^{N_{img} \times d}
\end{equation}
where \(E_{img}\) denotes the visual backbone and projection layers, \(N_{img}\) is the total number of visual tokens across all views, and \(d\) is the latent dimension of the transformer.

To preserve spatial information, two-dimensional positional embeddings are added to the projected image features. If a study contains fewer views than the maximum expected number of views, missing view slots are represented using padding tokens and ignored by the attention mask. This allows the model to handle variable-view studies while maintaining a consistent tensor structure.

\subsection{Text Embedding Model}
\label{subsec:text_embedding_model}

The text modality consists of pre-imaging clinical indication and history text. Post-interpretation report sections such as findings and impression are excluded because they may contain the target diagnosis and would therefore introduce label leakage.

The final implementation uses the radiology-domain CXR-BERT encoder microsoft/BiomedVLP-CXR-BERT-specialized to obtain text representations from the clinical indication and history text. This choice is suitable because CXR-BERT is trained on chest radiology language and image-text data, making it closer to MIMIC-CXR report language than a general biomedical encoder. To reduce computational cost, the text encoder is frozen and text embeddings are precomputed before multimodal training. A trainable projection layer maps the text representation into the shared fusion dimension.

For each text stream, the model uses the final \texttt{[CLS]} representation as a compact sentence-level embedding with dimensionality 768, passed through a learned linear projection into the multimodal fusion width. The projection expands each text embedding into a small number of fusion tokens, controlled by \(n_{\text{text}}\). In the main experiments, \(n_{\text{text}}=2\), so each clinical text field contributes two learned text tokens of width \(d_{\text{model}}\). Segment embeddings are added so that the fusion module can distinguish current clinical indication, prior clinical indication, and prior report text.

This design preserves clinically relevant textual context while keeping the training pipeline feasible under limited GPU memory. However, freezing the text encoder also prevents task-specific adaptation, which is treated as a limitation in Subsection~\ref{subsec:discussion_limitations}.

\subsection{Vital Signs Encoder}
\label{subsec:method_vitals_encoder}

Structured vital signs provide numerical clinical context related to the patient's physiological state. The vital-sign vector includes variables such as temperature, heart rate, oxygen saturation, respiratory rate, systolic blood pressure, and diastolic blood pressure. Each variable is normalized using statistics computed from the training set.

Because clinical data often contains missing values, the model also uses a missingness mask. The normalized vital-sign vector and missingness mask are concatenated and passed through a small multilayer perceptron to produce a vital-sign token in the shared latent space:
\begin{equation}
    h_i^{vit} = \mathrm{MLP}_v([\tilde{V}_i; M_i]) \in \mathbb{R}^{1 \times d}
\end{equation}
where \([\cdot ; \cdot]\) denotes vector concatenation, and \(\mathrm{MLP}_v\) is a multi-layer perceptron projection network mapping the concatenated physiological vector to the latent dimension \(d\).

\subsection{Multimodal Token Construction}
\label{subsec:method_token_construction}

After modality-specific encoding, the model converts all inputs into tokens for transformer-based fusion. The current-study token sequence \(X_i^{cur}\) is constructed by concatenating the representations from the visual, textual, and physiological encoders:
\begin{equation}
    X_i^{cur} = [H_i^{img}; h_i^{txt}; h_i^{vit}] \in \mathbb{R}^{N_{cur} \times d}
\end{equation}
where \([ \cdot ; \cdot ]\) denotes sequence concatenation along the token dimension, \(h_i^{txt} \in \mathbb{R}^{n_{\text{text}} \times d}\) represents the projected text tokens, and \(N_{cur} = N_{img} + n_{\text{text}} + 1\) is the total number of target tokens.

If prior-study information exists, a corresponding memory stream token sequence \(X_i^{prior}\) is constructed similarly:
\begin{equation}
    X_i^{prior} = [H_i^{prior-img}; h_i^{prior-txt}; h_i^{prior-vit}; e_{\Delta t}] \in \mathbb{R}^{N_{prior} \times d}
\end{equation}
where \(H_i^{prior-img}\), \(h_i^{prior-txt}\), and \(h_i^{prior-vit}\) are the encoded visual, textual, and vital tokens of the historical prior examination, and \(e_{\Delta t}\) is a learned embedding representing the time gap \(\Delta t\) between the current and prior studies. The text sequence \(h_i^{prior-txt} \in \mathbb{R}^{2n_{\text{text}} \times d}\) contains the projected tokens from both the prior clinical indication and prior report text, leading to a total of \(N_{prior} = N_{prior-img} + 2n_{\text{text}} + 2\) prior tokens.

Segment embeddings tell the transformer which source each token came from after projection into the shared latent space. Positional embeddings are used for spatial image tokens, while temporal embeddings are used for prior-study tokens. Attention masks are applied to ignore padded views or unavailable modalities.

To make training feasible under limited GPU memory, the model uses compact token layouts rather than dense spatial maps. For standard self-attention, the computational and memory complexity scales quadratically with the sequence length \cite{vaswani2017attention}:
\begin{equation}
    \mathcal{O}(N^2 d)
\end{equation}
where \(N\) is the number of tokens and \(d\) is the hidden dimension. By projecting spatial visual grids into a reduced set of tokens \(N_{img}\), the attention cost is minimized, allowing multimodal fusion to run within hardware constraints.

\subsection{Prior-Aware Temporal Fusion}
\label{subsec:method_prior_fusion}

We incorporate prior-study information because clinicians compare current and previous examinations. The core mechanism of the fusion module is Multi-Head Attention (MHA), following the Transformer attention formulation \cite{vaswani2017attention}. For a query matrix \(Q\), key matrix \(K\), and value matrix \(V\), scaled dot-product attention is defined as:
\begin{equation}
    \mathrm{Attn}(Q,K,V) = \mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
\end{equation}
where $d_k$ is the key projection dimension.

The fusion module uses an asymmetric target-memory design. Current-study tokens form the target stream, while prior-study tokens form the memory stream. The target stream first undergoes self-attention to model interactions among current image, text, and vital-sign tokens:
\begin{equation}
    Z_i^{cur} = \mathrm{MHA}(X_i^{cur}, X_i^{cur}, X_i^{cur})
\end{equation}
It then attends to the prior-study memory stream through cross-attention:
\begin{equation}
    Z_i^{fused} = \mathrm{MHA}(Q=Z_i^{cur}, K=X_i^{prior}, V=X_i^{prior})
\end{equation}
This cross-attention design keeps the current study as the prediction target and uses the prior study only as supporting memory.

To represent temporal distance, we compute the time gap $\Delta t_i$ in days between the current study and the nearest previous chest X-ray study from the same patient. The time gap is then mapped into one of seven discrete buckets:
\begin{equation}
b_i = \mathrm{bucket}(\Delta t_i)
\end{equation}
\begin{equation}
e_{\Delta t_i} = E_{\Delta t}(b_i)
\end{equation}
where $E_{\Delta t}$ is a learned embedding lookup table.

The seven buckets are: $\leq 1$ day, 2--7 days, 8--30 days, 1--6 months, 6--12 months, 1--3 years, and $>3$ years. These buckets give finer resolution to recent priors, because acute follow-up studies often happen within days or weeks. Longer gaps are grouped more coarsely because very old priors are usually less directly comparable to the current study.

    \caption{Distribution of prior-study time-gap buckets. Most prior studies occur within the first week, while very long gaps over three years are rare. This supports the use of finer buckets for recent priors and coarser buckets for older priors.}

As shown in Figure~\ref{fig:time_gap_bucket}, most available priors are recent, especially within one week. The model therefore learns separate embeddings for short-term follow-up studies while still keeping compact representations for older priors.

When no prior study is available, the memory stream is replaced by a learned no-prior token. In this case, no time-gap bucket is used because there is no valid prior-study time difference. During training, stochastic prior dropout is applied with probability \(p_{drop}\). Let \(r_i \in \{0, 1\}\) be a random variable sampled as:
\begin{equation}
    r_i \sim \mathrm{Bernoulli}(1-p_{drop})
\end{equation}
The prior tokens are then scaled during training:
\begin{equation}
    X_i^{prior'} = r_i X_i^{prior}
\end{equation}
This regularization reduces, but cannot remove, shortcut risk by forcing the model to use both current-study and prior-study evidence.

The exploratory analysis in Section~\ref{subsec:eda_prior_gaps} (Figure~\ref{fig:temporal_label_dynamics}) shows that prior labels often persist between studies but can also change, particularly for common findings. The model therefore treats prior information as auxiliary context rather than a deterministic shortcut: time-gap embeddings allow the fusion module to weigh historical evidence according to temporal distance, while prior dropout ensures that the classification head remains functional when prior context is absent or unreliable.

\subsection{Classification Head and Training Objective}
\label{subsec:method_head_loss}

The fused token sequence is passed to a multi-label classification head. Instead of using one global pooled feature, the classifier uses learnable label queries inspired by ML-Decoder \cite{ridnik2023mldecoder}. Each label query attends to the fused multimodal tokens and produces one logit for one pathology class.

Let the learnable label queries be:
\begin{equation}
Q_{\text{label}} = [q_1, q_2, \ldots, q_C] \in \mathbb{R}^{C \times d},
\end{equation}
where $C=26$ is the number of target labels and $d$ is the fusion dimension. Each label query attends to the fused token sequence:
\begin{equation}
G_i = \mathrm{MHA}(Q_{\text{label}}, Z^{\text{fused}}_i, Z^{\text{fused}}_i) \in \mathbb{R}^{C \times d}.
\end{equation}
A class-specific projection maps each label representation to one logit:
\begin{equation}
z_i \in \mathbb{R}^{C}.
\end{equation}
The predicted probability for class $c$ is computed using a sigmoid function:
\begin{equation}
\hat{y}_{ic} = \sigma(z_{ic}) = \frac{1}{1 + e^{-z_{ic}}}.
\end{equation}

The model is trained with Asymmetric Loss (ASL) \cite{ridnik2021asymmetric}. ASL is suitable for long-tailed multi-label classification because it down-weights easy negative labels more strongly than positive labels. This helps prevent frequent negative labels from dominating training.

For one sample, the loss is:
\begin{equation}
\mathcal{L}_{ASL}
=
-\sum_{c=1}^{C}
\left[
y_c (1-p_c)^{\gamma_+}\log(p_c)
+
(1-y_c)(p_c^m)^{\gamma_-}\log(1-p_c^m)
\right],
\end{equation}
where $p_c=\hat{y}_{ic}$ is the predicted probability for class $c$. The shifted negative probability is:
\begin{equation}
p_c^m = \max(p_c - m, 0),
\end{equation}
where $m = 0.05$ is the negative margin. The parameters $\gamma_+$ and $\gamma_-$ control the focusing strength for positive and negative labels. In this work, we set $\gamma_+ = 1$ and $\gamma_- = 4$, satisfying $\gamma_- > \gamma_+$ so that easy negative labels receive less weight during training.

\subsection{Evaluation Metrics}
\label{subsec:method_evaluation_metrics}

We evaluate the model using mean Average Precision (mAP) and macro-averaged AUROC (mAUROC). The primary metric is mAP, following the CXR-LT-2023 evaluation setting \cite{holste2024cxrlt}. This metric is suitable for long-tailed multi-label classification because it measures ranking quality over the precision-recall curve for each class.

For each pathology class $c$, Average Precision (AP) is computed as:
\begin{equation}
AP_c = \sum_n (R_n - R_{n-1})P_n,
\end{equation}
where $P_n$ and $R_n$ are the precision and recall at the $n$-th threshold. The overall mAP is:
\begin{equation}
mAP = \frac{1}{C}\sum_{c=1}^{C} AP_c.
\end{equation}

We also report macro-averaged AUROC:
\begin{equation}
mAUROC = \frac{1}{C}\sum_{c=1}^{C} AUROC_c.
\end{equation}

To analyze long-tailed performance, we report metrics separately for head, medium, and tail label groups. These groups follow the label-frequency analysis described in Section~\ref{sec:eda}.

\subsection{Qualitative Attribution}
\label{subsec:method_qualitative_attribution}

For image attribution, we use Grad-CAM to produce class-specific heatmaps for selected predictions \cite{selvaraju2017gradcam}. Given a target class logit ($z_c$) and a convolutional feature map ($A^k$), Grad-CAM first computes the importance weight for channel $k$ as
\begin{equation}
    \alpha_k^c = \frac{1}{Z}\sum_i\sum_j \frac{\partial z_c}{\partial A_{ij}^k},
\end{equation}
where $Z$ is the number of spatial locations. The final heatmap is then computed as
\begin{equation}
    L_{\mathrm{Grad-CAM}}^c = \mathrm{ReLU}\left(\sum_k \alpha_k^c A^k\right).
\end{equation}
This heatmap highlights image regions that positively contribute to the selected class prediction. For the multi-label setting, we compute the attribution separately for each target finding using its pre-sigmoid logit.

For text attribution, we use gradient-based token saliency, following prior work on visualizing neural NLP models \cite{li2016visualizing_nlp}. Specifically, we compute gradient $\times$ embedding scores at the CXR-BERT token level. Since CXR-BERT uses a subword tokenizer, these tokens do not always correspond exactly to full words. For readability, the visualization renders tokenizer outputs as word-like text chips by removing subword continuation markers. Therefore, the highlighted text should be interpreted as token-level attribution rather than exact word-level attribution. For token $t$ with embedding $e_t$, the attribution score is computed as
\begin{equation}
    s_t = \left\| e_t \odot \frac{\partial z_c}{\partial e_t} \right\|_1.
\end{equation}
Tokens with larger scores have a stronger local influence on the selected class prediction. These scores are used only for qualitative inspection and are not treated as clinically validated explanations.

\subsection{Ablation Study Design}
\label{subsec:method_ablation_design}

Although this section describes the final model, the experiments also include ablations to measure the effect of each major design choice. The ablations compare image-only and multimodal inputs, prior-study context, time-gap embeddings, prior dropout, and engineered versus plain image channel encoding. The results are reported in Section~\ref{sec:results}.
